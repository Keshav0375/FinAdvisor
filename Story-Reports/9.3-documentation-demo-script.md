# 9.3 — Documentation + Demo Script

## Problem
The project needed user-facing documentation for setup, a structured demo walkthrough for presentation/recording, and a security threat model to demonstrate compliance awareness.

## Implementation
- **`README.md`** — Complete project README with:
  - Mermaid architecture diagram showing all services and data flow
  - Request flow text diagram (browser -> Next.js -> FastAPI -> Agent -> tools -> LLM)
  - Tech stack table (10 layers)
  - Key features list (RLS, citations, stale warnings, PII, eval gate, fallback)
  - Mock users table (4 profiles with tier/jurisdiction/licenses)
  - Quick start guide (7-step setup)
  - Key commands reference
  - Project structure overview
  - Documentation links
- **`docs/demo_script.md`** — Step-by-step demo walkthrough with 5 scenes:
  - Scene 1: Product suitability (Sarah Chen, US senior, multi-tool queries)
  - Scene 2: Jurisdiction enforcement (Alex Kim, EU associate, cross-border blocking)
  - Scene 3: Refusal and escalation (tier denial, PII refusal, scope limits)
  - Scene 4 (optional): Stale document warning with orange badge
  - Scene 5 (optional): LangFuse tracing walkthrough
  - Recording tips for consistent framing
- **`docs/threat_model.md`** — Security considerations across 7 areas:
  - Data access control (RLS, tier hierarchy)
  - PII protection (ingest + output redaction)
  - Authentication/authorization (mock + production notes)
  - LLM security (prompt injection, data exfiltration, fallback chain)
  - Infrastructure security (secrets, network, containers)
  - Eval and quality gates (threshold enforcement, stale handling)
  - Compliance considerations (audit trail, regulatory refs, data residency)

## Decisions
- Used mermaid for the architecture diagram rather than an image file — renders natively on GitHub, no external hosting needed.
- Demo script includes 3 main scenes (matching the verification requirement) plus 2 optional bonus scenes for deeper demonstrations.
- Threat model structured around STRIDE-like categories but adapted to RAG-specific threats (prompt injection, citation fabrication, tier bypass).
