# 7.1 — Layout + UserSwitcher Component

## Problem
The frontend needed a base layout with a header and a way for users to switch between the 4 mock advisor personas, with the selection persisting across page refreshes.

## Implementation
- **`frontend/src/lib/users.ts`** — UserProfile interface and USERS array with 4 mock users synced with backend mock_users.py (sarah_chen, alex_kim, james_wright, priya_sharma)
- **`frontend/src/hooks/useUser.ts`** — Zustand store with localStorage persistence for current user selection
- **`frontend/src/components/UserSwitcher.tsx`** — Client component with select dropdown showing name, tier, jurisdictions; displays tier level and licenses
- **`frontend/src/components/Header.tsx`** — Client component with FinAdvisor branding + UserSwitcher
- **`frontend/src/app/layout.tsx`** — Updated with Header, metadata, and max-w-5xl content container
- **`frontend/src/app/page.tsx`** — Placeholder for chat interface (task 7.2)

## Decisions
- Used zustand instead of React context for user state — zustand is already a project dependency and provides simpler persistence patterns.
- Header and UserSwitcher are both "use client" components since they read from zustand/localStorage.
- User profiles use camelCase in TypeScript (tierLevel) while backend uses snake_case (tier_level) — the API boundary will handle conversion.

## Verification
- ESLint: 0 warnings or errors
- `npm run build`: compiles successfully
- Dev server at localhost:3000: header renders with FinAdvisor title and user dropdown, default user sarah_chen loaded
