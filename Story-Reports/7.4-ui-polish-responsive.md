# 7.4 — UI Polish + Responsive Design

## Problem
The raw chat interface needed professional finance-industry styling, mobile responsiveness, loading state indicators, and visual polish before the frontend phase could be considered complete.

## Implementation
- **`globals.css`** — Neutral gray background (#f8f9fa), Geist system font, custom thin scrollbar styling
- **`Header.tsx`** — Sticky header with backdrop blur, FA logo badge (blue rounded square), "Compliance-Aware Wealth Advisor" subtitle
- **`UserSwitcher.tsx`** — Compact select dropdown, tier badge and jurisdiction labels hidden on mobile (sm:flex)
- **`ChatWindow.tsx`** — Empty state with chat bubble SVG icon and description text, animated spinner on send button during streaming, scrollbar-thin class, responsive padding (px-4 → sm:px-6)
- **`MessageBubble.tsx`** — Rounded-2xl bubbles with shadow-sm, tool calls separated with border-b, blue streaming cursor (rounded-full), animation classes for message appearance
- **`CitationPanel.tsx`** — Shadow-lg popup with SVG icons for regulatory reference and close button, formatted dates (MMM DD, YYYY)
- **`tailwind.config.ts`** — Custom fade-in and slide-in-from-bottom keyframes and animation utilities

## Decisions
- Used sticky header with backdrop-blur for a modern, floating feel.
- Hidden tier/jurisdiction details on mobile to keep the header compact — the select dropdown still shows the essential user info.
- Custom scrollbar styling only on Webkit (Chrome, Safari, Edge) — Firefox uses default thin scrollbar.
- Animation classes referenced in JSX (`animate-in fade-in`) rely on Tailwind's animation extend.
