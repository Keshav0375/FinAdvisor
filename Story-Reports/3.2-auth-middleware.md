# 3.2 — Auth Middleware Integration

## Problem
Need a running FastAPI application with the auth dependency wired in, per-request structured logging (user_id, tier, jurisdictions), and basic API routes. This is the foundation all future endpoints build on.

## Implementation
- `backend/src/main.py` — App factory with:
  - `create_app()` function (FastAPI with lifespan context manager)
  - `request_logging_middleware` — logs every request with method, path, status, duration_ms, user_id via structlog
  - Module-level `app = create_app()` for uvicorn
- `backend/src/api/__init__.py` — API package init
- `backend/src/api/health.py` — Two endpoints:
  - `GET /api/health` — public, returns `{"status": "ok"}`
  - `GET /api/me` — auth-protected, returns current user's claims
- `backend/tests/test_auth.py` — Updated to 5 tests using real app:
  - valid user → 200 + correct claims
  - unknown user → 401
  - missing header → 401
  - health endpoint → 200
  - request logging fires (smoke test)

## Decisions
- Used `app.middleware("http")` decorator for request logging rather than a Starlette `BaseHTTPMiddleware` subclass — simpler, fewer import hops.
- Health endpoint is public (no auth) — needed for Docker healthchecks and load balancer probes.
- `/api/me` endpoint doubles as auth verification — frontend can call it to confirm user identity.
- Middleware logs `user_id` from header even on unauthenticated requests (as "anonymous") — useful for debugging.
