# 1.2 — SQLAlchemy ORM Models

## Problem
Need Python ORM models that map to the PostgreSQL schema so the application can interact with documents, chunks, and suitability rules in a type-safe way.

## Implementation
- `backend/src/db/__init__.py` — Package marker.
- `backend/src/db/models.py` — Three models using SQLAlchemy 2.x DeclarativeBase:
  - `Document`: all columns from documents table, relationship to Chunk (cascade delete-orphan)
  - `Chunk`: includes `Vector(1024)` column via pgvector-python, FK to documents, relationship back to Document
  - `SuitabilityRule`: all columns from suitability_rules table

## Decisions
- Used `Mapped[list[float]]` for the embedding column with `Vector(1024)` type — pgvector-python handles serialization.
- Used `uuid.uuid4` as Python-side default rather than relying on `gen_random_uuid()` — allows setting IDs before flush.
- `created_at` mapped as `Mapped[datetime | None]` with `default=None` — lets the DB handle the default via `now()` in schema.sql.
- Bidirectional relationship between Document ↔ Chunk for convenient navigation in both directions.
