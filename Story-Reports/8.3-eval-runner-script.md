# 8.3 — Eval Runner Script

## Problem
The golden Q&A eval set (8.2) needed an automated runner that executes each entry against the agent, scores responses with an LLM-as-judge, and produces structured results with threshold gating for CI/CD integration.

## Implementation
- **`backend/scripts/run_eval.py`** — Standalone async eval runner:
  - Loads `golden_qa.json` eval set and `judge_prompt.txt`
  - Streams each question to the backend API via httpx SSE client
  - Collects response text, tool calls, and tool results from SSE events
  - Sends question + response + retrieved chunks to LLM-as-judge (Claude) for scoring
  - Checks expected tool calls and must_not_contain content violations
  - Outputs `eval_results.json` with per-question scores, by-category aggregates, and overall averages
  - `--compare-baseline` flag for delta comparison against previous results
  - `--limit N` flag for quick partial runs
  - `print_summary()` displays formatted table and gates on thresholds (citation_accuracy >= 95%, faithfulness >= 90%)
- **`backend/tests/test_eval_runner.py`** — 14 unit tests covering:
  - File loading (eval set, judge prompt, users)
  - Tool call expectation matching (match, missing, empty)
  - Content violation checking (clean, violation, case-insensitive)
  - Judge response parsing (valid JSON, malformed JSON, API error)
  - Full golden_qa.json structure validation (50 entries, all fields, all categories)
  - Judge prompt content validation

## Decisions
- Used httpx streaming client against the live API rather than direct agent instantiation — avoids duplicating DI wiring and ensures eval tests the full stack including auth, RLS, and PII redaction.
- JSON extraction from judge output uses `find`/`rfind` for `{`/`}` to handle cases where the judge wraps JSON in markdown or prose.
- Threshold gating in `print_summary()` calls `sys.exit(1)` on failure — makes the script usable as a CI gate step.
- `--limit` flag enables quick smoke tests during development without running all 50 entries.

## Verification

- Script loads and shows help: `python scripts/run_eval.py --help`
