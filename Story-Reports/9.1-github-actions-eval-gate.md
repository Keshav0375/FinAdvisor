# 9.1 — GitHub Actions Eval Gate

## Problem
The eval runner (8.3) and golden Q&A set (8.2) existed but weren't wired into CI. Without an automated gate, degraded system prompts or broken citation logic could ship to production unchecked.

## Implementation
- **`.github/workflows/eval-gate.yml`** — Activated from placeholder:
  - `lint-and-test` job: Python 3.12 setup, `pip install -e ".[dev]"`, ruff check, ruff format --check, mypy src/, pytest tests/ -x
  - `eval-gate` job (depends on lint-and-test): pgvector service container, DB setup + seed, starts backend with uvicorn, runs `run_eval.py` against live API, checks thresholds (citation_accuracy >= 95%, faithfulness >= 90%), uploads results as artifact
  - Triggered on pull_request to main

## Decisions
- Added explicit `lint-and-test` prerequisite job per ARCHITECTURE.md spec rather than combining into one job — eval is expensive (LLM calls), so fast-fail on lint/type/test errors first.
- Updated threshold check to read from `aggregate` key (matching `run_eval.py` output format where scores are nested under `results['aggregate']`, not at top level as in the original placeholder).
- Added `if: always()` to artifact upload step so results are available for debugging even when thresholds fail.
- Backend started with `uvicorn` in background before running eval — the eval runner hits the live API via httpx, testing the full stack.

## Verification
- YAML syntax validated with `yaml.safe_load()`
- Quality gate: ruff clean, mypy 0 errors, pytest 97 passed / 25 skipped (DB tests skip without Postgres — pre-existing)
- Full CI verification deferred (requires GitHub PR to trigger the workflow)
