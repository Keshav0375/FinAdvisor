# 1.4 — Database Integration Test

## Problem
Need to prove that RLS policies actually enforce access control correctly — a tier-3 user must not see tier-4 chunks, and an EU user must not see US chunks. This is the system's security foundation.

## Implementation
- `backend/tests/conftest.py` — Shared test infrastructure:
  - `can_connect_to_db()` + `requires_db` marker: skips DB tests gracefully when Postgres unavailable
  - `async_engine` fixture: creates tables from ORM models, tears down after
  - `db_session` fixture: provides an AsyncSession
  - `seeded_session` fixture: inserts 9 chunks (US tiers 1-4, EU tiers 1-3, UK tiers 1,4), enables RLS, creates chunk_visibility policy
  - Mock user constants: SARAH_CHEN (US/tier-3), ALEX_KIM (EU/tier-1), PRIYA_SHARMA (US+EU/tier-2)
- `backend/tests/test_rls.py` — Three policy enforcement tests:
  - `test_tier3_user_sees_tier_1_2_3_not_4`: Sarah sees US tier 1-3, not 4
  - `test_eu_only_user_sees_eu_chunks_not_us`: Alex sees only EU tier-1 (1 chunk)
  - `test_combined_tier_and_jurisdiction_filter`: Priya sees US+EU tier 1-2 (4 chunks)
- Updated `test_rls_context.py` to use conftest fixtures and `requires_db` marker.

## Decisions
- Used `can_connect_to_db()` at module load time to determine if tests should skip — avoids slow timeouts during test collection.
- Seed data uses raw SQL INSERT (not ORM) because the seeded_session also creates RLS policy in the same session — ORM would require separate handling.
- 9 chunks cover all the permutations needed: multiple jurisdictions, multiple tiers, including edge cases (tier-4, multi-jurisdiction user).
- RLS policy is created in the test fixture (not relying on schema.sql) so tests are self-contained.

## Verification
- `pytest tests/ -v` — 5 tests collected, all 5 skip properly (no DB), 0 failures.
- `ruff check .` — all checks passed.
- `mypy src/` — success, no issues.
- Full verification requires Docker (`docker compose up postgres` + `pytest tests/test_rls.py -v`).
