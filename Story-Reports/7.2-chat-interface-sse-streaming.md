# 7.2 — Chat Interface + SSE Streaming

## Problem
The frontend needed a complete chat interface that sends messages to the backend, receives SSE-streamed responses, and displays them progressively — including tool call indicators, error handling, and streaming text animation.

## Implementation
- **`frontend/src/lib/api.ts`** — `streamChat()` function using fetch + ReadableStream for SSE parsing; sends X-User-Id header with each request
- **`frontend/src/hooks/useChat.ts`** — Chat state hook managing messages array, streaming state, tool calls, citations; processes SSE events (message, tool, citation, error, done)
- **`frontend/src/components/ChatWindow.tsx`** — Full chat container with message list, input form, auto-scroll, and clear button; disabled input during streaming
- **`frontend/src/components/MessageBubble.tsx`** — User (blue) and assistant (white) message bubbles; shows tool call indicators (green dot + tool name), error text (red), and streaming content
- **`frontend/src/components/StreamingText.tsx`** — Text renderer with animated cursor (pulsing gray bar) during active streaming
- **`frontend/src/app/page.tsx`** — Updated to render ChatWindow as "use client" page

## Decisions
- Used native `fetch` + `ReadableStream` instead of eventsource-parser — simpler for POST-based SSE and avoids the EventSource limitation of GET-only requests.
- Messages are immutable-updated via `prev.map()` pattern for React state correctness.
- Tool calls appear as compact indicators above the response text, not as separate messages.
- useChat is a standalone hook (not zustand) since chat state is ephemeral and doesn't need cross-component sharing beyond the ChatWindow tree.
