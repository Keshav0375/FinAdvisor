# 0.3 — Docker Compose Foundation

## Problem
Need a local dev environment that runs all services (database, backend, frontend, LLM gateway, observability) with a single command, matching the architecture's service map.

## Implementation
- `docker-compose.yml` — 8 services: postgres (pgvector:pg16 with healthcheck), backend, frontend, litellm, kong, langfuse, langfuse-db, redis. 3 named volumes for data persistence. Dependency ordering with health conditions.
- `docker-compose.override.yml` — Dev overrides: volume mounts for hot-reload on backend (src/, tests/) and frontend (src/), debug log level.
- `.env.example` — All env vars documented with placeholder values. Covers LLM keys, embeddings, database, PII mode, Kong, LangFuse, auth.
- `backend/db/schema.sql` — Placeholder (enables pgvector + pgcrypto extensions). Full schema comes in task 1.1.

## Decisions
- Added healthchecks on postgres and langfuse-db so dependent services don't start before DB is ready.
- Used `condition: service_healthy` for backend → postgres dependency (prevents connection errors on startup).
- Put redis in even though it's optional (rate-limiting via Kong) — avoids config changes later.
- Placeholder schema.sql created now so the volume mount in docker-compose doesn't point to a missing file.

## Verification
- YAML validated via Python yaml.safe_load (both files parse correctly).
- Programmatic check confirms all 8 required services present.
- Docker Desktop not installed on this machine — `docker compose up postgres` deferred to when Docker is available.
