# 8.2 — Golden Q&A Eval Set

## Problem
The eval system needed a comprehensive set of golden Q&A pairs to test agent behavior across product suitability, jurisdiction enforcement, refusal correctness, citation accuracy, and stale document handling. Without this, there's no way to gate deployments on quality.

## Implementation
- **`data/eval/golden_qa.json`** — 50 Q&A entries across 5 categories:
  - Product suitability (15): correct recommendations, regulatory citations, risk matching
  - Jurisdiction scoping (10): cross-border boundary enforcement (US/EU/UK/dual)
  - Refusal scenarios (10): tier access denial, PII requests, scope refusal, override attempts
  - Citation accuracy (10): correct [N] citation mapping, regulatory ref verification
  - Stale doc handling (5): staleness warnings on all 4 stale-marked corpus documents
- **`data/eval/judge_prompt.txt`** — LLM-as-judge prompt scoring faithfulness, citation_accuracy, and refusal_correctness on 0.0–1.0 scale with JSON-only output.

## Decisions
- All 4 user profiles used with realistic distribution (sarah_chen: 15, alex_kim: 15, james_wright: 10, priya_sharma: 10) to ensure every tier/jurisdiction combination is tested.
- Questions grounded in actual corpus products (Meridian fund names, real regulatory refs from generate_corpus.py) rather than generic finance questions — ensures eval tests real retrieval, not hallucination.
- Stale doc category tests all 4 stale-marked documents from the corpus (Balanced Income Portfolio, European Small Cap, UK Equity Income Trust, Alternative Investment Qualification Rule).
- Judge prompt includes clarification that refusal_correctness should score 1.0 for correctly answering non-refusal questions (not just for correct refusals).

## Verification
- JSON validates: 50 entries, all required fields present, all IDs unique
- Category distribution: 15+10+10+10+5 = 50
- Quality gate: ruff clean, mypy clean, pytest 106 passed / 2 skipped
- Human approval: granted
