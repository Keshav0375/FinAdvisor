# 7.3 — Citation System

## Problem
The agent's responses contain `[N]` citation markers that need to be rendered as interactive inline badges. Clicking a badge should expand a panel showing the source document title, regulatory reference, review date, and a stale warning for outdated documents.

## Implementation
- **`frontend/src/lib/parseCitations.ts`** — Regex parser that splits response text into `TextSegment` and `CitationSegment` arrays at `[N]` boundaries
- **`frontend/src/components/CitationInline.tsx`** — Clickable blue badge showing citation number; toggles CitationPanel on click; gray if citation metadata unavailable
- **`frontend/src/components/CitationPanel.tsx`** — Expandable panel showing source title, regulatory ref, review date, and close button
- **`frontend/src/components/StaleBadge.tsx`** — Orange "STALE" badge rendered when `lastReviewedAt` is older than 12 months
- **`frontend/src/components/MessageBubble.tsx`** — Updated to parse citations in assistant messages and render CitationInline components inline with text

## Decisions
- Used `RegExp.exec()` loop instead of `String.matchAll()` — the tsconfig target doesn't support `downlevelIteration` and matchAll returns an iterator.
- Citations are positioned absolutely relative to the badge for popup-style expansion without disrupting text flow.
- Citation data is mapped by index for O(1) lookup when rendering inline badges.
