# 0.5 — CI Workflow Scaffold

## Problem
Need automated quality checks on every PR to prevent regressions. Also need the eval gate structure ready for Phase 8 activation.

## Implementation
- `.github/workflows/ci.yml` — Two parallel jobs: backend (ruff check, ruff format --check, mypy, pytest) and frontend (npm ci, lint, tsc --noEmit, build). Triggered on PR + push to main.
- `.github/workflows/eval-gate.yml` — Full eval gate structure from ARCHITECTURE.md Section 14, disabled with `if: false`. Includes pgvector service container, schema setup, eval runner, threshold checks, artifact upload.
- `backend/tests/__init__.py` — Empty package marker so pytest discovers the tests directory.

## Decisions
- Added `push` trigger to ci.yml (not just PR) so main branch always shows green status.
- Used `if: false` on eval-gate rather than commenting out steps — keeps the YAML valid and parseable while clearly marking it inactive.
- Frontend CI includes `tsc --noEmit` separately from `npm run build` to catch type errors even if build passes (build may skip some checks).
- Used `npm ci` in CI (clean install from lockfile) rather than `npm install`.

## Verification
- Both YAML files parse correctly via yaml.safe_load.
- Structure validated: ci.yml has 2 jobs (backend, frontend) with correct step sequences.
- eval-gate.yml has postgres service container, all required steps.
