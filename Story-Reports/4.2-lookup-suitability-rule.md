# 4.2 — Tool: lookup_suitability_rule

## Problem
The agent needs a tool to look up firm suitability rules by product category and client risk profile, filtered by the advisor's jurisdiction and tier level.

## Implementation
- `backend/src/agent/tools/lookup_suitability.py` — Pydantic models (SuitabilityInput, SuitabilityRuleResult, SuitabilityOutput) + `lookup_suitability_rule()` async function that queries the `suitability_rules` table with jurisdiction, tier, product_category, and risk_profile filters
- `backend/tests/test_lookup_suitability.py` — 5 tests with seeded rules: US user match, EU user match, multi-jurisdiction sees both, tier filter excludes high-tier rules, no match returns empty

## Decisions
- No RLS on `suitability_rules` table — filtering is done in the WHERE clause using `jurisdiction = ANY(:jurisdictions)` and `min_tier_required <= :tier`. RLS is only on `chunks` (the vector search table).
- The tool takes an explicit `UserClaims` parameter for jurisdiction/tier filtering, keeping it testable without needing to SET ROLE.
- Seeded test data is cleaned up with `DELETE FROM suitability_rules` in fixture teardown.
