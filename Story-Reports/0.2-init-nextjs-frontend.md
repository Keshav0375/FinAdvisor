# 0.2 — Initialize Next.js Frontend Project

## Problem
Need a frontend scaffold with TypeScript, Tailwind CSS, and the App Router pattern before building chat UI components. Must proxy API calls to the FastAPI backend.

## Implementation
- `frontend/` — Scaffolded via `create-next-app@14` with TypeScript, Tailwind, ESLint, App Router, `src/` directory.
- `frontend/package.json` — Added zustand (state management) and eventsource-parser (SSE streaming).
- `frontend/next.config.mjs` — Rewrites `/api/*` to `http://localhost:8000/api/*` for backend proxy.
- `frontend/src/app/page.tsx` — Minimal placeholder page (replaced default template).
- `frontend/tsconfig.json` — `strict: true` confirmed (default from scaffold).

## Decisions
- Used `next.config.mjs` (ESM) since create-next-app@14 generates that format by default.
- Used `rewrites` instead of `redirects` so the proxy is invisible to the browser (no CORS issues in dev).
- Kept the default Tailwind config untouched — will customize in Phase 7 UI polish task.
- Replaced template page.tsx with a minimal one — the real chat UI comes in task 7.2.

## Verification
- `npm run dev` — starts successfully ("Ready in 4s" on port 3000).
- `npm run build` — compiles cleanly, generates static pages.
- `npm run lint` — no ESLint warnings or errors.
