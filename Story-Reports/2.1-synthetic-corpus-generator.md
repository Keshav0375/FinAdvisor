# 2.1 — Synthetic Corpus Generator

## Problem
The RAG pipeline needs realistic financial documents to ingest, embed, and retrieve. Manually writing 50 compliance documents isn't feasible — need automated generation with proper metadata for RLS testing (tier/jurisdiction tags), stale-date detection, and PII redaction testing.

## Implementation
- `backend/scripts/generate_corpus.py` — Main generator script:
  - 50 DOCUMENT_SPECS with full metadata (title, doc_type, jurisdiction, tier_required, product_category, risk_level, regulatory_ref, stale flag, pii flag)
  - Uses Claude API (claude-sonnet-4-20250514) with tailored system prompts per doc type
  - Outputs JSON files to `data/corpus/{doc_type}/` directories
  - Supports `--dry-run` mode (prints plan, no API calls)
  - Cost estimation logged at startup (~$0.50-$1.00 for 50 docs)
  - structlog for all logging (progress, word counts, timing)
- `data/corpus/product_factsheets/` — 20 product factsheets (US/EU/UK, tiers 1-4, all risk levels)
- `data/corpus/suitability_rules/` — 10 suitability rules (jurisdiction-specific regulations)
- `data/corpus/compliance_memos/` — 10 compliance memos (internal, various jurisdictions)
- `data/corpus/jurisdiction_disclosures/` — 10 jurisdiction disclosures (US/EU/UK/APAC)

## Decisions
- Used Claude Sonnet (not Opus) for generation — sufficient quality at ~10x lower cost for bulk generation.
- Each spec has explicit metadata rather than asking the LLM to invent it — ensures deterministic tier/jurisdiction coverage for RLS testing.
- Documents are 396-541 words — realistic length for chunking (will produce 1-2 chunks per doc at 512-token window).
- PII embedded in privacy notices (phone numbers, emails) and some client documents — provides material for PII redaction pipeline testing.
- Stale dates (>12 months old) on select docs — enables stale-document detection testing.
- `--dry-run` allows reviewing the plan without incurring API costs — follows CLAUDE.md cost awareness rule.

## Verification
- `python scripts/generate_corpus.py` produced exactly 50 JSON files
- All files have valid JSON structure with required fields: title, doc_type, jurisdiction, tier_required, regulatory_ref, last_reviewed_at, content
- Tier coverage: tiers 1-4 represented across product factsheets
- Jurisdiction coverage: US, EU, UK, APAC
- PII confirmed in doc 49 (US Privacy Notice): phone `(555) 123-4567`, email `compliance@meridianwealth.com`
- Word counts: 396-541 words per document (within 300-800 target range)
- Quality gate: ruff check, ruff format, mypy, pytest all pass
