# 1.1 — PostgreSQL Schema File

## Problem
Need the database schema that implements pgvector storage with row-level security — the core differentiator that enforces access control at the database level rather than in application code.

## Implementation
- `backend/db/schema.sql` — Complete PostgreSQL 16 schema (19 statements):
  - `documents` table: source documents with doc_type, jurisdiction, tier_required, regulatory_ref, risk_level, last_reviewed_at
  - `chunks` table: embedded chunks with vector(1024), denormalized metadata from documents for RLS hot path
  - `suitability_rules` table: structured rules with product_category, client_risk_profile, jurisdiction
  - 4 indexes: ivfflat on embedding (cosine), btree on document_id, jurisdiction, tier_required
  - RLS: ENABLE + FORCE on chunks, chunk_visibility policy (tier + jurisdiction check via session vars)
  - Roles: finadvisor_service (bypasses RLS), finadvisor_app (subject to RLS)
  - Grants: appropriate per-role permissions on all tables

## Decisions
- Used DO block with conditional role creation to make schema idempotent (won't fail if roles already exist).
- Set ivfflat lists=10 (appropriate for ~500 chunks; ARCHITECTURE.md note says increase for larger corpus).
- Placed suitability_rules after GRANT statements for chunks/documents (execution order matters for role grants).
- RLS policy uses `current_setting('app.user_tier')` and `current_setting('app.user_jurisdictions')` — these must be SET before any query.

## Verification
- SQL parsed successfully via sqlparse (19 valid statements).
- Statement types verified: CREATE EXTENSION/TABLE/INDEX/POLICY, ALTER TABLE, DO block, GRANT.
- Full `psql -f schema.sql` verification requires Docker (not available in this environment).
