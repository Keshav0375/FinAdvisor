# 0.1 — Initialize Python Backend Project

## Problem
The project had no backend scaffold. Need a properly configured Python package with dependency management, type checking, linting, and environment configuration before any feature code can be written.

## Implementation
- `backend/pyproject.toml` — Package definition with all runtime deps (fastapi, sqlalchemy, anthropic, voyageai, langfuse, structlog, etc.) and dev deps (pytest, ruff, mypy, httpx). Includes ruff and mypy configuration inline.
- `backend/src/__init__.py` — Package marker.
- `backend/src/config.py` — `Settings` class (pydantic-settings) modeling all env vars from ARCHITECTURE.md Section 17: LLM keys, database URL, PII mode, Kong/LangFuse endpoints, JWT secret.

## Decisions
- Used `setuptools` as build backend (simple, universally supported) rather than hatch/flit.
- Put ruff + mypy config in `pyproject.toml` rather than separate config files to keep the root clean.
- Set mypy overrides to ignore missing imports for untyped third-party libs (voyageai, langfuse, google.cloud, jose, sse_starlette, pgvector).
- All Settings fields have defaults matching local dev values from ARCHITECTURE.md so the app can start without a .env file.

## Verification
- `pip install -e ".[dev]"` — installed successfully with all deps resolved.
- `python -c "from src.config import Settings"` — imports and instantiates cleanly.
- `ruff check .` — all checks passed.
- `mypy src/` — success, no issues found.
