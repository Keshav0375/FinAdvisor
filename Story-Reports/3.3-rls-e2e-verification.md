# 3.3 — End-to-end RLS Verification

## Problem
Need to prove that PostgreSQL Row-Level Security correctly filters vector similarity search results per user, with zero unauthorized chunk leakage across all 4 user personas.

## Implementation
- `backend/tests/test_rls_e2e.py` — 5 E2E tests against real ingested data (114 chunks):
  - sarah_chen (US/tier-3): sees 43 chunks, only US tiers 1-3
  - alex_kim (EU/tier-1): sees 10 chunks, only EU tier 1
  - james_wright (UK/tier-4): sees 26 chunks, all UK tiers
  - priya_sharma (US+EU/tier-2): sees 55 chunks, US+EU tiers 1-2
  - Cross-user verification: all 4 users get different result counts
- `backend/src/db/rls.py` — Added `SET ROLE finadvisor_app` to `set_rls_context()`, added `reset_rls_context()` helper
- `backend/tests/conftest.py` — Added JAMES_WRIGHT, `live_session` fixture, removed destructive `drop_all`, seed data cleanup with `__TEST_SEED__` prefix
- `backend/tests/test_rls.py` — Fixed to coexist with real data (content prefix filtering)

## Decisions
- Used `SET ROLE finadvisor_app` in `set_rls_context()` — the `finadvisor` user is a superuser that bypasses RLS entirely. Switching to the non-superuser app role is required for RLS enforcement.
- Used literal string interpolation for SET commands — asyncpg doesn't support parameterized queries for PostgreSQL SET statements. Safe because values come from validated Pydantic models.
- Removed `Base.metadata.drop_all` from `async_engine` fixture teardown — was destructive to real ingested data.
- E2E tests run against live data (read-only) while seeded tests use `__TEST_SEED__` prefix + per-test cleanup for isolation.

## Verification
- `pytest tests/ -v` — 25 passed, 2 skipped
- All 4 user scenarios pass with zero unauthorized chunk leakage
- `assert_zero_leakage()` checks every returned row satisfies tier and jurisdiction constraints
- Cross-user test confirms all users see different result sets
- Real data intact after test run: 50 documents, 114 chunks
