# 6.4 — Backend Integration Test Suite

## Problem
The backend had unit tests for individual components but lacked end-to-end integration tests that verify the full request → SSE stream → parsed events flow including citations, PII redaction, multi-tool dispatch, and error handling.

## Implementation
- **`backend/tests/conftest.py`** — Added `test_client` fixture (AsyncClient with ASGITransport) and `override_dependencies` fixture (yields app.dependency_overrides dict, clears on teardown)
- **`backend/tests/test_integration.py`** — 9 integration tests with reusable mock agent factories and SSE event parser

### Tests
1. `test_full_stream_with_citations` — Full flow: tool_call → tool_result → citation → text with [N] notation → done
2. `test_multi_tool_dispatch_stream` — Two sequential tool calls (search_firm_kb + lookup_suitability_rule)
3. `test_stream_error_event` — Agent max iterations error propagated as SSE error event
4. `test_pii_redacted_in_stream_output` — Email and phone redacted from LLM text output
5. `test_stream_always_ends_with_done` — Stream terminates with `event: done`
6. `test_conversation_id_accepted` — Optional conversation_id field accepted
7. `test_different_users_get_same_stream_format` — All 4 mock users produce valid SSE
8. `test_missing_message_returns_422` — Pydantic validation rejects empty body
9. `test_request_id_header_in_response` — X-Request-Id echoed via RequestContextMiddleware

## Decisions
- SSE parser uses `line.strip() == ""` for blank line detection to handle Windows `\r\n` line endings.
- Mock agents are factory functions (not fixtures) for flexible composition across tests.
- Used `override_dependencies` fixture pattern for automatic cleanup instead of manual try/finally.
