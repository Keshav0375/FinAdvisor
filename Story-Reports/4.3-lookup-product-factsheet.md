# 4.3 — Tool: lookup_product_factsheet

## Problem
The agent needs a tool to retrieve full product fact sheets by name, with results filtered by the advisor's jurisdiction and tier level.

## Implementation
- `backend/src/agent/tools/lookup_factsheet.py` — Pydantic models (FactsheetInput, FactsheetResult, FactsheetOutput) + `lookup_product_factsheet()` async function that queries the `documents` table with ILIKE fuzzy match on title, filtered by doc_type, jurisdiction, and tier
- `backend/tests/test_lookup_factsheet.py` — 6 integration tests against real ingested data: exact name match, partial match, case-insensitive, jurisdiction filtering (US vs EU see different results), tier filtering, no match

## Decisions
- Used `ILIKE` for case-insensitive fuzzy matching — more flexible than exact match, matches the "fuzzy match on title" requirement from TODO.md.
- Queries the `documents` table directly (not chunks) — factsheets are full documents, not vector-search results.
- No RLS on `documents` table for this query — filtering is done in WHERE clause with jurisdiction and tier checks, same pattern as lookup_suitability_rule.
- Tests run against real ingested data (live_session fixture) — no seed data needed since the corpus has 9 US and 6 EU product factsheets.
