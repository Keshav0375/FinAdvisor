# 0.4 — Git Hygiene + Linting Setup

## Problem
Need consistent code quality enforcement across both backend and frontend, and a comprehensive .gitignore to prevent committing artifacts, secrets, and build outputs.

## Implementation
- `.gitignore` — Rewrote to cover Python (pycache, venv, mypy/ruff caches), Node (node_modules, .next), Docker volumes, IDE files (.idea, .vscode), Terraform state, and env files.
- `backend/ruff.toml` — Dedicated config: line-length=100, target py312, isort with src as first-party. Removed duplicate from pyproject.toml.
- `backend/pyproject.toml` — Retained mypy config (strict, ignore_missing_imports for untyped libs) and pytest config.
- `frontend/.prettierrc.json` — Standard formatting: semi, double quotes, 2-space indent, trailing commas, 100 char width.
- `frontend/.eslintrc.json` — Extended with "prettier" to disable conflicting ESLint rules.

## Decisions
- Moved ruff config to standalone `ruff.toml` (task explicitly calls for it, and it's easier to find than buried in pyproject.toml).
- Chose double quotes in Prettier (matches Next.js/React convention from create-next-app output).
- Used `eslint-config-prettier` rather than `eslint-plugin-prettier` — disabling conflicts is preferred over running Prettier as an ESLint rule.

## Verification
- `ruff check .` — "All checks passed!"
- `mypy src/` — "Success: no issues found in 2 source files"
- `npm run lint` — "No ESLint warnings or errors"
