# 1.3 — Async Session Factory + RLS Helpers

## Problem
Need an async database connection layer and the critical `set_rls_context` function that sets PostgreSQL session variables before every query — this is what makes RLS enforcement work.

## Implementation
- `backend/src/db/session.py` — `build_engine(settings)` factory: creates async engine with connection pooling (pool_size=5, max_overflow=10), returns `async_sessionmaker[AsyncSession]`.
- `backend/src/db/rls.py` — `set_rls_context(session, user)`: executes `SET app.user_tier` and `SET app.user_jurisdictions` as parameterized SQL. Must be called before any query that touches chunks.
- `backend/src/auth/__init__.py` — Package marker.
- `backend/src/auth/models.py` — `UserClaims` Pydantic model (sub, name, tier, tier_level, jurisdictions, licenses).
- `backend/tests/test_rls_context.py` — Integration test: sets RLS context, verifies `current_setting()` returns correct values.

## Decisions
- Created `auth/models.py` now (even though task 3.1 will add more auth files) because `rls.py` directly depends on `UserClaims` — better to have the real type than a Protocol hack.
- Used parameterized queries (`:tier`, `:jurisdictions`) even for SET statements to prevent SQL injection.
- `expire_on_commit=False` in session factory — avoids lazy-load issues when accessing attributes after commit in async context.
- Pool size of 5 with overflow to 10 — reasonable for local dev, production can tune via config.
