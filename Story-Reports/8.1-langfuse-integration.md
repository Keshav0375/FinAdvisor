# 8.1 — LangFuse Integration

## Problem
The agent orchestrator had no observability — no way to trace query execution, measure LLM call latency, inspect tool call chains, or version the system prompt. Production debugging and eval scoring required a tracing backend.

## Implementation
- **`backend/src/observability/tracing.py`** — `TracingContext` class wrapping the LangFuse client. Noop-safe when LangFuse is `None`. Methods: `start_trace`, `start_generation`, `end_generation`, `start_span`, `end_span`, `update_trace`, `score`, `flush`. Also `get_prompt_from_langfuse()` for versioned system prompt retrieval with fallback.
- **`backend/src/agent/orchestrator.py`** — Refactored `FinAdvisorAgent` to use `TracingContext`: trace per query (with user/tier/jurisdiction metadata), generation per ReAct iteration, span per tool call. Prompt versioning: fetches from LangFuse first, falls back to local `get_system_prompt()`.
- **`langfuse/docker-compose.langfuse.yml`** — Standalone LangFuse compose: `langfuse/langfuse:2` on port 3030, Postgres 16 on port 5433, auto-init with org/project/keys (`pk-lf-finadvisor-dev` / `sk-lf-finadvisor-dev`), admin user seeded.
- **`backend/tests/test_tracing.py`** — 12 unit tests: noop safety, trace creation, generation lifecycle, span lifecycle, scoring, flush, prompt retrieval (success, fallback, error cases).

## Decisions
- Made `TracingContext` noop-safe (returns `None` when LangFuse is absent) so the agent works identically without observability infrastructure — no conditional checks scattered through orchestrator code.
- Standalone compose file rather than adding to the main `docker-compose.yml` — LangFuse is optional infrastructure, keeps the core compose lean.
- Auto-init keys in compose environment so `docker compose up` gives a working LangFuse instance with zero manual setup.
- Prompt versioning checks LangFuse first, falls back to local — enables A/B testing system prompts via LangFuse UI without code deploys.
