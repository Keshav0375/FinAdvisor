# 2.3 — Voyage AI Embeddings Client

## Problem
The ingest pipeline needs to embed document chunks as 1024-dimensional vectors for storage in pgvector. Need a production-ready client with rate limiting and retry logic to handle Voyage AI's API constraints.

## Implementation
- `backend/src/retrieval/__init__.py` — Package init
- `backend/src/retrieval/embeddings.py` — `VoyageEmbeddings` class:
  - Uses `voyageai.Client` SDK (sync, wrapped in `asyncio.to_thread` for async)
  - Model: `voyage-3` (1024 dimensions)
  - Batch size: 128 texts per request (Voyage API limit)
  - Rate limiting: `asyncio.Semaphore(3)` + 333ms interval between requests
  - Retry: 3 attempts with exponential backoff (1s, 2s, 4s)
  - Two public methods: `embed_texts(list[str])` and `embed_query(str)`
- `backend/tests/test_embeddings.py` — Integration tests:
  - Single sentence → 1024-dim float vector
  - Batch of 3 → 3 vectors of 1024 dims each
  - Skips gracefully when `VOYAGE_API_KEY` not in environment

## Decisions
- Used `asyncio.to_thread()` to wrap the sync voyageai SDK — avoids blocking the event loop without needing an async HTTP client wrapper.
- Rate limiter uses semaphore + sleep rather than a token bucket — simpler, sufficient for ingest workload (not high-concurrency).
- `input_type="document"` for ingest; `embed_query` will need `input_type="query"` for retrieval (future task can add this parameter).
- Tests use `pytest.mark.skipif` on env var presence — allows CI to skip without Voyage key, while local runs with `.env` loaded will exercise the real API.

## Verification
- `pytest tests/test_embeddings.py -v` with VOYAGE_API_KEY loaded — 2 tests pass (44s, includes API round-trips)
- Single sentence: returned exactly 1024 floats
- Batch of 3: returned 3 × 1024 vectors
- Quality gate: ruff, mypy, pytest all clean
