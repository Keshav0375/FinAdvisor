# 6.1 — App Factory + Health Endpoint

## Problem
The FastAPI backend needs a proper app factory with lifecycle management (DB pool, embeddings, LLM client, PII redactor, LangFuse) and dependency injection providers so the SSE streaming endpoint and agent can be wired up cleanly.

## Implementation
- `backend/src/main.py` — Enhanced `create_app()` factory with full `lifespan` context manager: initializes Settings, DB session factory (build_engine), VoyageEmbeddings client, PII redactor (create_redactor), AsyncAnthropic client (pointed at Kong/LiteLLM), and optional LangFuse (gracefully skips if no keys configured). Stores all on `app.state`. Request logging middleware unchanged.
- `backend/src/dependencies.py` — FastAPI DI providers: `get_settings`, `get_db_session` (yields AsyncSession from pool), `get_rls_session` (sets RLS context per user), `get_embeddings`, `get_redactor`, `get_llm_client`, `get_langfuse`, `get_agent` (assembles ToolRegistry + FinAdvisorAgent with all deps injected)
- `backend/src/api/router.py` — Main API router aggregator that includes health_router (and will include chat_router in 6.2)
- `backend/Dockerfile` — Based on python:3.12-slim, installs deps from pyproject.toml, runs uvicorn on port 8000
- `backend/src/api/health.py` — Already existed from task 3.2 (GET /health, GET /me)

## Decisions
- LLM client points to Kong (`settings.kong_url/v1`) with `litellm_master_key` as API key — matches ARCHITECTURE.md Section 8 ("backend's Anthropic client points to Kong, not directly to Anthropic")
- LangFuse is optional (None = no-op) — avoids breaking the app when LangFuse isn't configured locally
- `get_rls_session` is a separate dependency from `get_db_session` — only endpoints that need user-scoped data use it, while admin/health endpoints use the plain session
- `get_agent` composes ToolRegistry + FinAdvisorAgent per-request — each request gets its own session and tool registry, preventing cross-request state leakage
