# 6.2 — SSE Streaming Endpoint

## Problem
The backend needed an SSE streaming endpoint so the frontend can receive real-time agent responses (tool calls, citations, text) as they happen, with PII redacted from output before reaching the client.

## Implementation
- **`backend/src/api/schemas.py`** — Pydantic models: ChatRequest, Citation, ChatResponse
- **`backend/src/api/chat.py`** — POST /api/chat/stream endpoint using sse-starlette EventSourceResponse; `_format_sse_event` maps StreamEvent types to SSE event format with PII redaction on text events
- **`backend/src/api/router.py`** — Updated to include chat_router alongside health_router
- **`backend/tests/test_chat_stream.py`** — 7 tests covering event formatting, PII redaction, and full endpoint integration

## Decisions
- Used `dependency_overrides` pattern for integration tests instead of `patch()` — the app's lifespan initializes state on `app.state`, and test-client instantiation without Docker services would fail. Overriding `get_agent` and `get_redactor` dependencies cleanly bypasses lifespan requirements.
- PII redaction applied only to text events (not tool_call/tool_result) since tool data is internal agent state and redaction there could break tool execution.
- Citation events forwarded using `model_dump_json()` for zero-copy serialization.

## Verification
- Quality gate passed: ruff (0 errors), mypy (0 errors), pytest (74 passed, 2 skipped)
- Unit tests verify each SSE event type formats correctly
- Integration test confirms full POST → SSE stream with tool/tool_result/message/done events
- Auth test confirms 401 without X-User-Id header
